Tijdens het derde jaar heb ik voor 3 maanden stage gelopen bij De Lijn. Ik werkte op de agile manier, dus in een team van 7 met een scrum master en stand-up meetings elke dag. Maar ook op het einde van de 2 weken lang durende sprint end of sprints meetings.

Tijdens deze stage kreeg ik als opdracht automatische testen schrijven voor een android native app van De Lijn. Deze testen zijn geschreven met webdriverio en appium. Ze zouden bij elke nieuwe versie van de app gedraaid moeten worden.

Zij zullen checken of de app blijft werken zoals we verwachten. Voor meer informatie over mijn stage kan u in het realisatieverslag kijken. Het project bevatte meerdere fases zoals het opzoeken van informatie over de verschillende frameworks en een proof of concept maken met deze informatie. En dan als tenslotte de testen zelf schrijven.

Vroeger werd de app manueel getest bij elke nieuwe release dit nam enorm veel tijd in beslag. Momenteel kan je mijn testen gebruiken deze duren in het totaal een klein uurtje waardoor er veel sneller getest wordt.

Tryhackme

tryhackme is een website waarop je kan leren pentesten. Je kan verschillende paden volgen met de nodige uitleg om deze paden af te werken. Zo heb ik al verschillende paden afgewerkt over vanalles.

Dit pad ging over de basis van offensive pentesting. Zoals de eerste stappen dat je altijd doet bij het krijgen van een pentesting opdracht. Zoals het scannen van open poorten, vulnerability scanning. Maar ook web apps attacks zoals SQL injection.
Als tweede ging je echt vulnerabilities zoeken zo heb ik leren werken met verschillende databases met vulnerabilities. Daarna waren het meer specifieke opdrachten over vanalles. Meer informatie kan je terug vinden op de website van tryhackme zelf.

Uit dit heb ik enorm veel geleerd over het beveiligen van projecten en ook hoe je problemen kunt terugvinden. Dit hebben we bijvoorbeeld gebruikt bij het testen van andere groepen hun project php. Zo kon je in groepen hun website een php shell doorsturen en zo toegang krijgen tot de server. Dit hebben we dan gemeld.

Project 4.0

Project 4.0 was een opdracht voor Vito. Wij hebben een web app gemaakt die aan leeftijdsherkenning kan doen bij mais planten. Aan de rechterkant kan u de stappen zien die een foto doorgaat bij deze applicatie.

Zo gaat een foto getrokken met de gsm naar de nodejs back-end. Daarna stuurt deze back-end de foto door naar een blob storage van azure. Deze wordt dan gebruikt om de AI bij te leren. Deze AI stuurt dan de leeftijd van de plant terug.

Deze wordt dan getoond om de angular front-end. Nadien wordt het antwoord en de link naar de foto ook opgeslagen op de MySQL database zodat we nadien de antwoorden kunnen herbekijken.

Al deze aparte delen waren docker containers gedraaid als kubernetes cluster in okteto cloud. Zo hadden ze elk toegang tot elkaar en was het makkelijk op te zetten met maar enkele stappen.

In dit project heb ik samen met een collega zowel de angular front-end als de nodejs back-end gemaakt. Maar ik heb ook alles van het hosten op de cloud gedaan. Dus de juiste docker images aanmaken voor zowel de AI als de front- en back-end.

java app in kubernetes

Voor het vak advanced programming topics moesten we een project maken waarbij 1 edge service communiceerde met 3 api's en nadien een front-end met deze edge service communiceerde. Dit werd allemaal gedraaid aan de hand van kubernetes pods op okteto cloud.

Voor dit project heb ik de bonen api gemaakt maar ook aan de edge service meegeholpen. Zo heb ik vanalles wat meegepikt.
